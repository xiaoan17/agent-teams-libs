# PRD Authoring Skill

- Clarify ambiguous requirements with the user before writing.
- Capture goals, user stories, priorities, and acceptance criteria.
- Keep each requirement testable and unambiguous.
- Hand the PRD to the CTO for breakdown and to UI/UX for design.
